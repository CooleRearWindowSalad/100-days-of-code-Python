## Commit Style Guide

This project follows a structured commit message convention:

- `feat:` new exercise or feature
- `fix:` bug fixes
- `refactor:` code improvements
- `docs:` documentation updates

**Examples:**
```
feat: added day01 challenge solutions
fix: corrected package name in Challenge01
docs: updated README project structure
refactor: improved variable naming in day02
```
